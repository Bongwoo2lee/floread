리다이렉션에 대해 알려드릴게요
리다이렉션은 입력과 출력의 방향을 다른 곳으로 바꾸는 것입니다.
리눅스의 표준 입력은 키보드로부터 입력받고 표준 출력은 터미널에 출력됩니다. 

먼저 표준 입력을 파일로 하는 법에 대해 알려드리겠습니다.
명령어] < [파일]
sort < sorting.txt

그 다음 표준 출력을 파일로하는 것입니다. 
> 새로 저장 
새로 저장되므로 기존 파일이 있는지 확인하고 해야함
>> 뒤에 내용 추가
ls -al > file
이런식으로 하시면 출력문이 file에 저장됩니다.
cat file 하시면 출력되는 것을 볼수 있습니다.   

그리고 아까 했던 명령어인 cat명령어는 파일의 내용을 출력하는 명령어인데 
원래 cat file은 파일을 출력하는데 
cat > file하시면 입력한 내용이 파일에 저장
컨트롤d하시면 저장됩니다.

그 다음에 파이프에 대해 알려드리겠습니다.
파이프는 입출력의 방향을 프로세스로 바꿀 수 있습니다.
앞 명령어의 결과를 뒤에 나오는 명령어의 입력으로 처리하기 위해 사용
예를들어 
ls | grep file하면 ls의 출력이 grep의 입력으로 들어가고 grep명령어는 뒤에 입력된 문자열을 포함한 행을 출력하는 명령어인데 file이름이 들어간 파일만 출력하는 문장입니다.
